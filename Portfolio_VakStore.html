Project description (brief)

Vak’s Store is a web app that supports a community-run trading platform and an experimental in-game economy for the Vintage Story server TOPS (The Official Public Server). It is designed for TOPS players who want to trade without having to coordinate real-time meetups.

It exists to address three persistent problems: barter friction (scheduling + trust overhead), Rusty Gear (RG) inflation (RG can be mass-produced by mob farms), and price opacity (a lack of shared reference prices).

  • Main Store: build buy/sell carts and submit admin-approved trade requests (asynchronous trading via in-game mailboxes).
  • EW currency: ledger-based currency issued only when players sell to the store, destroyed when they buy from the store and via OCM admin fees.
  • OCM marketplace: player-to-player listings with peg-based pricing and admin/service fee model.
  • Public observability: Bank Dashboard + Price History charts, visible without login.
  • Insurance: optional metals-backed reserve mechanism for EW.
Role / team: Solo development by Vak (coding, design, documentation). “Arctic Toast” contributed penetration testing and security feedback.

System overview (high-level features)

Main Store (admin approvals)

  • Cart building + calculators
  • Submit buy/sell requests
  • Admin executes and issues receipts

EW mechanics

  • Issuance: only when players sell items to the store
  • Destruction: store purchases + OCM admin fee + (optional) metal-withdraw nullification
  • Transaction history: immutable balance change log

OCM marketplace

  • Player listings + filters
  • Multi-peg pricing and payment options
  • Admin-service fee model (10%)

Observability + tools

  • Bank Dashboard (circulation, issuance/destroyed, backing, wealth distribution)
  • Price History (prices + velocity charts)
  • Leaderboards + Work Pay Rates
